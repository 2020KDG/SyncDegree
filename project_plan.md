# [프로젝트 기획서] 에듀-로컬업 (Edu-LocalUp)
**지역 산업 종사자를 위한 AI 기반 맞춤형 마이크로디그리 추천 서비스**

---

## 1. 서론

### 1-1. 과제 설계의 필요성
최근 '글로컬대학30' 사업의 확산에 따라 대학은 지역 사회의 평생교육 거점으로서의 역할이 강조되고 있습니다. 특히 농업, 제조업 등 지역 기반 산업 종사자들은 급격한 디지털 전환(DX) 환경에 직면해 있으나, 방대한 대학 교육 과정 중 자신에게 꼭 필요한 실무 교육을 찾기 어렵다는 문제점이 있습니다. 본 프로젝트는 이러한 간극을 메우기 위해 대학의 교육 자원을 AI 기술로 분석하여 지역 주민들에게 실질적인 성장을 지원하는 맞춤형 교육 로드맵을 제공하고자 합니다.

### 1-2. 과제의 개요
"에듀-로컬업"은 대학의 공개된 강의계획서(Syllabus) 데이터를 수집하고, 이를 RAG(검색 증강 생성) 기술로 분석하여 사용자 맞춤형 커리큘럼을 제안하는 MVP(Minimum Viable Product) 구현을 목표로 합니다. 단순한 키워드 검색을 넘어, LLM이 사용자의 직무 맥락을 이해하고 적절한 강의를 선별하여 '입문-중급-심화'의 3단계 마이크로디그리 형태로 제안합니다.

---

## 2. 본론

### 2-1. 주요 수행 내용
1.  **데이터 크롤링**: 대상 대학의 웹사이트에서 최신 강의계획서 데이터를 실시간/배치 방식으로 수집하는 크롤러 구현.
2.  **벡터 DB 구축**: 수집된 텍스트 데이터를 임베딩하여 고성능 검색이 가능한 ChromaDB 기반 벡터 스토어 구축.
3.  **지능형 RAG 파이프라인**: 
    - 사용자의 입력을 분석하여 검색 쿼리 확장.
    - 검색된 후보군 중 LLM이 강의의 맥락을 판단하여 최적의 강의 선별 및 단계별 배치.
4.  **프론트엔드 UI 개발**: 사용자가 직무와 목표를 입력하고 결과를 직관적으로 확인할 수 있는 웹 인터페이스 구현.

### 2-2. 수행 방법 (기술 스택)
-   **Backend**: Python FastAPI
-   **Orchestration**: LangChain
-   **AI 모델**: Google Gemini API (1.5 Pro/Flash)
-   **Database**: ChromaDB (Vector Store)
-   **Frontend**: HTML5, JavaScript, Tailwind CSS

### 2-3. 수행 일정 (8주)
| 주차 | 주요 과업 | 상세 내용 |
| :--- | :--- | :--- |
| **1-2주** | **요구사항 및 크롤링** | 요구사항 확정, 대학별 크롤링 모듈 개발 및 **ChromaDB 기초 인프라 구축(완료)** |
| **3-4주** | **데이터 및 DB 구축** | 수집 데이터 정제, 임베딩 및 ChromaDB 데이터 적재 로직 구현 |
| **5-6주** | **RAG 및 API 개발** | LLM 기반 검색 확장 및 선별 로직 구현, FastAPI 연동 |
| **7-8주** | **UI 구현 및 통합** | Tailwind CSS 기반 프론트엔드 개발 및 전체 시스템 통합 테스트 |

---

## 3. 결론

### 3-1. 최종 목표
특정 기관의 권한 승인 없이도 공개된 대학 오픈 데이터와 크롤링만으로 구동 가능한 독립적인 시스템을 구축합니다. 이를 통해 어떤 대학이나 지역 사회에서도 즉시 도입 가능한 범용적인 교육 추천 솔루션을 실현합니다.

### 3-2. 기대 효과 및 활용 방안
-   **기대 효과**: 지역 산업 종사자의 교육 접근성 향상 및 대학-지역 산업 간의 실질적인 산학협력 모델 제시.
-   **한계점 극복**: 데이터가 부족한 신산업 분야의 경우, LLM의 지식 베이스를 활용한 가이드라인 생성을 병행하여 추천의 공백을 최소화합니다.
-   **활용 방안**: 지자체 평생교육 포털과의 연동 또는 대학 내 마이크로디그리 과정 설계 지원 도구로 활용 가능.
